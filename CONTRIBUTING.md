## 開発環境のセットアップ

詳細なセットアップ手順は [README.md](README.md) を参照してください。

### 必要な環境

- **Rust**: 1.89.0以降
- **Bun**: 最新版
- **macOS**: 14.2以降（現在はmacOSのみ対応）
- **Vosk APIライブラリ**: `libvosk.dylib`
- **Voskモデル**: 日本語または英語モデル

### クイックスタート

```bash
# リポジトリをクローン
git clone https://github.com/your-username/MemoBreeze.git
cd MemoBreeze

# 依存関係をインストール
bun install

# Vosk APIライブラリをセットアップ（README.mdを参照）
# Voskモデルをセットアップ（README.mdを参照）

# 開発サーバーを起動
bun run tauri dev
```

## コーディング規約

### TypeScript/React

- **フォーマッター・リンター**: Biome を使用（`bun run format` / `bun run lint` / `bun run check`）
- **命名規則**:
  - コンポーネント: PascalCase（例: `MainControlCard`）
  - フック: camelCaseで`use`プレフィックス（例: `useTranscriptionListener`）
  - 変数/関数: camelCase
  - 定数: UPPER_SNAKE_CASE
- **パフォーマンス**:
  - 配列/オブジェクト生成には`useMemo`を使用
  - イベントハンドラーには`useCallback`を使用
  - Zustandストアからはプリミティブ値のみを購読
- **ファイル構成**:
  - 機能ごとに`src/features/`配下に配置
  - 再利用可能なコンポーネントは`src/components/`
  - カスタムフックは`src/hooks/`

### Rust

- **フォーマッター**: `cargo fmt`を使用
- **リンター**: `cargo clippy`を実行してワーニングを解消
- **命名規則**: Rustの標準規約に従う
  - 型: PascalCase
  - 変数/関数: snake_case
  - 定数: SCREAMING_SNAKE_CASE
- **エラーハンドリング**: `Result<T, E>`を適切に使用

### コミット前のチェック

```bash
# フロントエンド: フォーマット・リント・インポート整理
bun run check

# Rust: フォーマットとリント（-D warnings で警告をエラー扱い）
bun run format:rust
bun run lint:rust
```

## コミットメッセージ

明確で一貫性のあるコミットメッセージを心がけてください。

### フォーマット

```
<type>: <subject>

<body>

<footer>
```

### Type

- `feat`: 新機能
- `fix`: バグ修正
- `docs`: ドキュメントのみの変更
- `style`: コードの意味に影響しない変更（空白、フォーマットなど）
- `refactor`: バグ修正や機能追加ではないコードの変更
- `perf`: パフォーマンス改善
- `test`: テストの追加や修正
- `chore`: ビルドプロセスやツールの変更

### 例

```
feat: システム音声のキャプチャ機能を追加

macOSのCoreAudioを使用してシステム音声をキャプチャする機能を実装。
これにより、Zoom会議などの音声も文字起こし可能になる。

Closes #123
```

## プルリクエスト

1. **フォークとブランチ作成**
   ```bash
   git checkout -b feature/your-feature-name
   ```

2. **変更をコミット**
   - 上記のコミットメッセージ規約に従う
   - 小さく論理的な単位でコミット

3. **テスト**
   - 新機能には適切なテストを追加
   - 既存のテストが通ることを確認

4. **プッシュ**
   ```bash
   git push origin feature/your-feature-name
   ```

5. **プルリクエストを作成**
   - わかりやすいタイトルと説明を記載
   - 関連するイシュー番号を記載（`Closes #123`など）
   - スクリーンショットや動画があれば追加

### プルリクエストのチェックリスト

- [ ] コードがフォーマットされている（`bun run format`, `bun run format:rust`）
- [ ] リンターのワーニングがない（`bun run lint`, `bun run lint:rust`）
- [ ] フロントエンドのチェックが通る（`bun run check`）
- [ ] 既存の機能が壊れていない
- [ ] ドキュメントが更新されている（必要な場合）
- [ ] コミットメッセージが規約に従っている

## イシューの報告

### バグ報告

以下の情報を含めてください：

- **環境**: OS、Rustバージョン、Bunバージョン
- **再現手順**: バグを再現するための具体的な手順
- **期待される動作**: 何が起こるべきか
- **実際の動作**: 実際に何が起こったか
- **スクリーンショット**: 可能であれば
- **エラーログ**: コンソールやログファイルのエラー

### 機能リクエスト

以下の情報を含めてください：

- **ユースケース**: どのような場面で使いたいか
- **提案する解決策**: どのように実装するか（アイデアがあれば）
- **代替案**: 他に考えられる方法
- **追加情報**: その他の関連情報
